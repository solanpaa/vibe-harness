Vibe Harness — 2 State Machines

Task (agent process) + Workflow Run (orchestrator). Reviews, proposals, parallel groups = data records managed by workflow actions.

Task = agent process lifecycle. 7 states, 12 transitions.
• cancelled = user-initiated abort (distinct from failed = error)
• CANCEL available from any non-terminal state
• Pause synchronization: Workflow PAUSE cascades to Task PAUSE and vice versa

Workflow Run = central orchestrator. 10 states, 30 transitions.

TASK_COMPLETED decision point (guards top-to-bottom):
  1. [isSplitStage] → awaiting_split_review
  2. [isAutoAdvance] → stay running (advance stage)
  3. (default) → awaiting_review + create review

Unified APPROVE: single event, guard [isLastStage] decides → finalizing vs → running (next stage)
Merge conflict recovery: finalizing → MERGE_CONFLICT → awaiting_review (user resolves, then re-approves)
Pause race: paused also handles TASK_COMPLETED/TASK_FAILED (task finished mid-pause)
Stale event protection: TASK_COMPLETED carries taskId, validated against context.currentTaskId
Data records updated as side effects: Review status, Proposal status, Parallel Group status

Only 2 machines: Task and Workflow Run. Reviews, proposals, parallel groups = data records updated by workflow actions.

1. Simple task (1-stage workflow)

TASK pending → provisioning → running → COMPLETE → completed WORKFLOW TASK_COMPLETED → awaiting_review (review created) ↳ User approves WORKFLOW APPROVE [isLastStage] → finalizing (review → "approved") WORKFLOW FINALIZE → completed ✓

2. Multi-stage workflow (plan → implement → review → fix)

TASK plan → COMPLETE → completed WORKFLOW TASK_COMPLETED → awaiting_review (plan review) WORKFLOW APPROVE → running (stage: "implement") TASK implement → COMPLETE → completed WORKFLOW TASK_COMPLETED [isAutoAdvance] → running (stage: "review") TASK review → COMPLETE → completed WORKFLOW TASK_COMPLETED [isAutoAdvance] → running (stage: "fix") TASK fix → COMPLETE → completed WORKFLOW TASK_COMPLETED → awaiting_review (fix review) WORKFLOW APPROVE [isLastStage] → finalizing → FINALIZE → completed ✓

3. Request changes (rerun cycle)

TASK COMPLETE → completed WORKFLOW TASK_COMPLETED → awaiting_review WORKFLOW REQUEST_CHANGES → running (review → "changes_requested", rerun task spawned) TASK rerun → COMPLETE → completed WORKFLOW TASK_COMPLETED → awaiting_review (round 2) ↳ Cycle repeats until APPROVE

4. Merge conflict recovery

WORKFLOW APPROVE [isLastStage] → finalizing ↳ finalizeAndMerge() hits git conflict WORKFLOW MERGE_CONFLICT → awaiting_review (conflict review created) ↳ User resolves and approves WORKFLOW APPROVE [isLastStage] → finalizing → FINALIZE → completed ✓ ↳ OR user sends back to agent WORKFLOW REQUEST_CHANGES → running (agent fixes conflicts)

5. Split → parallel → consolidate

TASK plan → COMPLETE → completed WORKFLOW TASK_COMPLETED → awaiting_review WORKFLOW SPLIT → running (review → "approved", stage: "split") TASK split agent → creates proposals → COMPLETE → completed WORKFLOW TASK_COMPLETED [isSplitStage] → awaiting_split_review ↳ User reviews proposals (can REQUEST_CHANGES to re-run split) WORKFLOW LAUNCH_PROPOSALS → running_parallel ↳ Parallel group created, child workflows launched ↳ Children run independently (each = Task + Workflow pair) ↳ All children done WORKFLOW CONSOLIDATE [allChildrenDone] → completed ✓

6. Pause / Resume

WORKFLOW PAUSE → paused (action: pauseCurrentTask) TASK PAUSE → paused (agent stopped) ... later ... WORKFLOW RESUME → running (action: resumeCurrentTask) TASK RESUME → running (agent restarted)

7. Cancellation

WORKFLOW CANCEL → cancelled (action: cancelCurrentTask / cancelChildWorkflows) TASK CANCEL → cancelled (agent terminated, sandbox cleaned up) ↳ Distinct from FAIL — user-initiated abort, not an error

8. Task failure

TASK FAIL → failed (non-zero exit / crash) WORKFLOW TASK_FAILED → failed ✗
